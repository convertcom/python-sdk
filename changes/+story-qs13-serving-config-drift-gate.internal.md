Add a build-time serving-config drift gate at parity with the JS/PHP/Ruby SDKs (qs-13). The backend serving pipeline now regenerates compile-time-only PEP 561 `.pyi` TypedDict stubs (`src/convert_sdk/_generated/serving_config.pyi`, shipped in the wheel, never imported at runtime) and opens an auto-PR on every serving-schema change. A `mypy --strict` contract probe (`typecheck/serving_config_contract.py`, never shipped) binds the fields the SDK depends on via literal-key subscription, so a renamed/removed serving field fails CI with `[typeddict-item]`. No new runtime or dev dependency — `httpx` stays the only runtime dependency; `TypedDict`/`Literal` are stdlib. Catches field **name** drift; type-domain and spec-vs-reality drift remain a documented follow-up.
