Close 24 cross-SDK parity gaps and correctness bugs surfaced during the dev-branch adversarial review.

**Rule operators (`evaluation/rules.py`)** now match the JavaScript SDK's `Comparisons` class. `equals`/`matches`/`equalsNumber` lowercase both sides and special-case array (`testAgainst in array`) and mapping (`str(testAgainst) in keys`). `contains`/`startsWith`/`endsWith` lowercase both sides; `contains` returns `True` for whitespace-only `testAgainst`. `regexMatches` is case-insensitive. `less`/`lessEqual` coerce numeric strings via a JS-equivalent `isNumeric` check and return `False` on type mismatch instead of raising. The new `isIn` operator (split by `|`, lowercase `testAgainst`) is implemented. Unknown operators return `False` instead of falling through to a `==` shortcut.

**Bucketing (`evaluation/bucketing.py`)** strict-equals `variation.status == "running"` (matching the JS `VariationStatuses` enum), and a missing `traffic_allocation` defaults to `100%` (matching JS) instead of the previous `0%`. Variations with `status="active"` (which JS would skip) are no longer admitted.

**Experience qualification (`evaluation/experiences.py`)** drops the parity-breaking `experience.status` and `audience.status` filters and adds `archived_experiences` support (matching JS `data-manager.ts`). The environment field is read as singular `experience.environment` to match JS, replacing the prior plural `experience.environments` list. Diagnostic reason codes change `experience_inactive` → `experience_archived` for the archived-list rejection.

**Feature variables (`evaluation/features.py`, `context.py`)** default to `type_cast=False` for parity with JS, which surfaces variable values verbatim from the config. The Python-only type-coercion remains available as opt-in (`type_cast=True`) and is now safe (malformed `int("1.5")` returns the frozen original instead of raising).

**Transport adapter (`adapters/transport/httpx_transport.py`)** holds a long-lived `httpx.Client` instead of constructing a fresh one per request — auto-refresh at 5-minute cadence no longer pays a TLS handshake per tick. The new `HttpxTransport.close()` releases the lazily-created client. URL path segments (`sdk_key`, `account_id`, `project_id`) are URL-encoded to prevent path traversal. User-supplied `headers` can no longer override the SDK's `Authorization` or `Content-Type`.

**Config loader (`config_loader/loader.py`)** chains the underlying `httpx.HTTPError` / `JSONDecodeError` via `from exc` (was `from None`) so operator tracebacks survive incidents. Already-typed `ConfigLoadError` is re-raised verbatim.

**Validators (`config_loader/validators.py`)** type-check the optional list-shaped collections (`experiences`, `features`, `goals`, `audiences`, `segments`, `archived_experiences`) at the boundary and reject string-typed payloads.

**Normalizer (`config_loader/normalizer.py`)** populates `setdefault("segments", ())` and `setdefault("archived_experiences", ())` so missing keys are first-class instead of "works by coincidence."

**Refresh worker (`config_loader/refresh.py`)** outer guard catches `BaseException` (was `Exception`) so a logging filter that calls `sys.exit` cannot kill the daemon thread silently. `RefresherStatus` timestamps are now timezone-aware `datetime` instances (was `float` POSIX times) for consistency with `LifecycleEventPayload.occurred_at`.

**Public API**: `Core.snapshot` raises `InitializationError` instead of `RuntimeError` (so `except ConvertSDKError` catches readiness failures). `Core.refresh_now(wait=False)` returns `None` instead of a meaningless `True`. `RefreshConfig.interval_seconds < 1.0` is rejected (sub-second polls would hammer the upstream). `__init__.py` now exports `RefresherStatus`, `EventBus`, `Transport`, `ConvertSDKError`, and `TrackingDeliveryError`.

**Tracking queue (`tracking/queue.py`)** raises a typed `TrackingDeliveryError` on partial-batch failure (carries `delivered_event_count` / `delivered_batch_count` / `remaining_event_count`; underlying transport error available via `__cause__`). Poison-pill batches that fail to serialise are quarantined with a `tracking.delivery.quarantined` diagnostic and removed from `_pending` instead of crashing every subsequent `release()` call.

**Parity-fixture generator (`scripts/generate_parity_fixtures.py`)** records the JS-SDK package version and git commit SHA in every fixture's `source` block, so a future regenerate-and-diff CI hook can detect drift between the recorded outputs and the pinned upstream.
