Add opt-in automatic config refresh (`SDKConfig.refresh=RefreshConfig(...)`). A daemon thread on `Core` periodically reloads the project config through the existing `Transport`, applies the new `ConfigSnapshot` through an atomic attribute swap, and reports `refresh.start`/`success`/`skipped`/`fail` events through `convert_sdk.diagnostics`. With the default `refresh=None` the SDK behaves byte-for-byte identical to the MVP. Adds `Core.close()`, `Core.refresh_now()`, and context-manager support for graceful shutdown.
