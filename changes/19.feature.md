Add opt-in automatic config refresh (`SDKConfig.refresh=RefreshConfig(...)`). A daemon thread on `Core` periodically reloads the project config through the existing `Transport`, applies the new `ConfigSnapshot` through an atomic attribute swap, refreshes the `TrackingQueue`'s `account_id`/`project_id` so subsequent conversions attribute correctly (matching the JS SDK's `ApiManager.setData()` behaviour), fires the new `LifecycleEvent.CONFIG_UPDATED` lifecycle event (analog of JS `SystemEvents.CONFIG_UPDATED`), and reports `refresh.start`/`success`/`skipped`/`fail` events through `convert_sdk.diagnostics`. `RefreshConfig` validates its fields at construction (rejects negative intervals, `factor < 1`, `max < initial`, and similar misconfigurations through `ConfigValidationError`). The worker thread is wrapped in an outer exception guard that emits `refresh.worker_crashed` rather than dying silently. With the default `refresh=None` the SDK behaves byte-for-byte identical to the MVP. Adds `Core.close()`, `Core.refresh_now()`, and context-manager support for graceful shutdown.
