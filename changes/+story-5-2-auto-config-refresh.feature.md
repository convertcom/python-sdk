Add opt-in automatic config refresh (`SDKConfig.refresh=RefreshConfig(...)`, FR31). A long-running remote (`sdk_key`) instance can now periodically re-fetch config on a daemon thread and atomically swap the immutable snapshot in place, with exponential backoff on transient failures, a `LifecycleEvent.CONFIG_UPDATED` event on each successful swap, and tracking attribution re-pointed to the refreshed project. Refresh is **off by default** — `refresh=None` preserves MVP behavior byte-for-byte (no daemon thread, no events, no added cost). Concurrency and backoff parameters are ratified in `docs/adr/0001-config-refresh-concurrency-and-backoff.md`.
